title:HTML和CSS学习总结

html5的上一个版本的标签（全兼容）
<!DOCTYPE html> ->版本声明，这个必须要，如果不写在ie6会进入怪异模式
盒模型（margin+border+padding+content）
怪异模式：margin+content
在css3里面有个属性可以强制进入怪异模式

标签：
1.双标签和单标签
双标签：
1）需要嵌套（内容）
单标签：
1）不需要嵌套

标签的类型：
1.块标签
独占一行，默认宽度百分百，可以设置宽高/margin
主要标签：p,div,h1-6,ul,ol,li,hr,br,form
2.行标签
在一行显示，内容撑开宽高，不可以设置宽高（除了input,img）、不能设置纵向margin
主要标签：a,img,input,strot,var….
3.特殊
table,tr,td
4.组合标签
ul>li,ol>li,table>tr>td,form>input

标签的嵌套：
1.组合标签必须按照嵌套规则来
2.块标签可以嵌套块标签和行标签（除了p）
3.行标签只能嵌套行标签（除了a）

seo:搜索引擎优化，sem:搜索引擎营销
pr值：网页的等级 黑帽(利用搜索引擎的漏洞提高排名)、灰帽(研究搜索引擎的漏洞同是回避搜索引擎的处罚规则)、白帽（利用关键字的优化，结构优化。。。）

关键字的优化，主要就是对标签的选择：
title(head标签里面)

h1-h2
strong
img(alt(替换文本),title(鼠标移入的提示内容))
a：外链，内链 （）
外链：友情链接，通过别的网站来提高自己的等级，从而达到排名的提升

表格：
cellspacing:合并单元格边距
colspan:合并列
rowspan:合并行

form表单：
action:提交的地址
method:提交数据的方式
1)get:安全性低，运行速度快
2)post:安全性高，运行速度慢

input:
text:文本框
password:密码框
checkbox:复选框
radio:单选框
submit:提交按钮
button:按钮（需要配合value,不写会出问题）
file:文件
reset:重置按钮！基本没用

textarea:多行文本
select：菜单
option

lable:跟form控件联系起来（写样式）

css:
引入方式：
1.外部-》整站
2.头部-》访问量（点击）高
3.标签-》js和后台
4.@import url();-》在css中引入另外一个css

选择器：
1.标签名，2.类，3.id，4.后代，5.组合，6.群组，7.伪类, 8.全局（*）

a标签的伪类：
link:连接平常的状态
visited:连接被访问过之后
hover:鼠标放到连接上的时候
active:连接被按下的时候

选择器的优先级：
范围越小，优先级越高
id->100

a#div1 .c->111

css基础设置：
font-family->字体设置
font-size->字体大小
font-style->字体样式
font-weight->字体粗细（bold）

color->颜色
text-indent->文本缩进
text-align-》水平居中
text-de…->修饰线
line-height->行高
word-spacing-》字间距

background-color

盒模型：
margin(外边距),padding(内边距)，border(边框)

display:
none：隐藏
block：转换为块元素
inline-block:转换为行标签，但可以设置宽高
table:转换为table
table-cell：转换为单元格

overflow:hidden;scroll

absolute:在需要层叠的时候，然后需要移动的对象就用它（也可以作为参考对象）

z-index：改变堆叠顺序

使用场景：左右布局

CSS Hack大致有3种表现形式，CSS类内部Hack、选择器Hack以及HTML头部引用(if IE)Hack，CSS Hack主要针对类内部Hack：比如 IE6能识别下划线”“和星号” “，IE7能识别星号” “，但不能识别下划线”“，而firefox两个都不能认识。等等
选择器Hack：比如 IE6能识别html .class{}，IE7能识别+html .class{}或者*:first-child+html .class{}。等等
HTML头部引用(if IE)Hack：针对所有IE：<![endif]–>，针对IE6及以下版本：<![endif]–>，这类Hack不仅对CSS生效，对写在判断语句里面的所有代码都会生效

水平居中的方式:
text-algin:center;(针对里面的内联元素)
margin:0 auto;(针对自身)
position:absolute;

垂直居中的方式:

1.line-height; 只限于一行文字
2.display:table;
将上一级的盒子display:table;然后将需要移动的盒子display:table-cell 然后利用vertical-align:middle是来垂直居中
vertical-align: top; vertical-align: bottom;可以在任意内联元素使用.
3.position:absolute;在已知自身宽高的情况下,先上下移动50%,再用margin移动自身宽高的一半.
div{
width: 50px;
height: 50px;
background-color: yellow;
position: absolute;
left: 50%;
top: 50%;
margin-left: -25px;
margin-top:-25px;
}
4.position:absolute;在不知道自身宽高的情况下配合translate使用.先上下移动50%,再用translate移动自身宽高的50%.
div{
position: absolute;
left: 50%;
top: 50%;
-webkit-transform: translate(-50%,-50%);
-ms-transform: translate(-50%,-50%);
-o-transform: translate(-50%,-50%);
transform: translate(-50%,-50%);
}
5.”绝对居中术”

http://blog.csdn.net/freshlover/article/details/11579669
我们经常用margin:0 auto来实现水平居中，而一直认为margin:auto不能实现垂直居中……实际上，实现垂直居中仅需要声明元素高度和下面的CSS:
[css] view plain copy

  • .Absolute-Center {
  • margin: auto;
  • position: absolute;
  • top: 0; left: 0; bottom: 0; right: 0;
  • }

我不是这种实现方法的第一人，可能这只是非常常见的一种小技术，我斗胆将其命名为绝对居中(Absolute Centering)，虽然如此，但是大多数讨论垂直居中的文章却从来不提这种方法，直到我最近浏览《How to Center Anything WithCSS》这篇文章的评论时候才发现这种用法。在评论列表中Simon和Priit都提及了此方法。

6.弹性盒子;只限于移动端
给父级加
div{
display:-webkit-box;
box-align:center;
}

box-align：start | end | center子元素的垂直对齐方式
box-pack：start | end | center 子元素的水平对齐方式
